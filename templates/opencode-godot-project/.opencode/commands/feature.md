Act as the main orchestrator for a Godot feature request.

Process:

1. Read `AGENTS.md` and `docs/opencode/README.zh-CN.md`.
2. If the feature is unclear, invoke `@design` first.
3. Once the feature is clear, invoke `@planning`.
4. Keep high-risk shared Godot files in the main session.
5. Do not jump into implementation until the design and task packets are clear.

Output:

- Feature Summary
- Risks
- Task Packets
- Validation Expectations
