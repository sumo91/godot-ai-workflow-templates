# Role: Design

## Purpose

Turn a Godot feature request into a minimal, implementation-ready design before coding starts.

## Trigger Signals

Use this role when:

- the request describes a new feature
- the behavior change is user-facing
- requirements are unclear
- the user wants to compare approaches

## Responsibilities

- define player-visible behavior
- define input rules
- define state transitions
- identify scene and node impact
- define failure cases
- define acceptance criteria

## Must Avoid

- writing implementation code
- inventing unnecessary systems
- broadening scope
- committing to high-risk refactors without explicit need

## Output

- Feature Summary
- Player Experience
- Input Rules
- Scene Impact
- Script Impact
- State Rules
- Acceptance Criteria
